# Merge ff-change + propose Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `ff-change` 和 `propose` 的 skill 主体合并为同一份指令文本，保留两个命令入口（`/opsx:ff` 和 `/opsx:propose`），消除 ~200 行重复 prompt。

**Architecture:** 合并后 `propose.ts` 保留完整实现（skill + 两个 command），`ff-change.ts` 的 skill template 改为委托给 propose 的指令主体（即两者共享同一段 `instructions`），`ff-change.ts` 的 command template 保持独立入口描述。两文件的导出函数签名不变，调用方无感知。

**Tech Stack:** TypeScript, Vitest

---

## 差异分析（合并前）

| 位置 | ff-change | propose | 差异 |
|------|-----------|---------|------|
| skill.description | 快速创建产出物（无产出物列表） | 列出 proposal/design/tasks 三产出物 | propose 更具体 |
| skill.instructions 开头 | `快速完成产出物创建 - 一次性生成...` | `提案新变更 - 一步创建变更并生成所有产出物。\n\n我将创建一个包含...` | propose 有 preamble 列出产出物类型 |
| 步骤 1-4 | 完全相同 | 完全相同 | ✅ |
| 步骤 5（最终状态） | 完全相同 | 完全相同 | ✅ |
| 输出 - 结尾提示 | `运行 /opsx:apply 或要求我实现` | `运行 /opsx:apply 或要求我实现以开始处理任务` | 微小措辞差异 |
| 产出物创建指南 | propose 多一句 `使用 template 作为输出文件的结构 - 填充其各个部分` | propose 版本 | propose 版本更完整 |
| 护栏 - 已存在变更 | `建议继续处理该变更`（无询问） | `询问用户是否要继续它或创建新的` | propose 版本更合理 |
| command.description | `一键创建变更并生成实现所需的所有产出物` | `提案新变更 - 一步创建并生成所有产出物` | 各保留自己的 |
| command 输入说明 | `/opsx:ff 之后的参数...` | `/opsx:propose 之后的参数...` | 各保留自己的 |
| command 输出提示 | `运行 /opsx:apply 以开始实现` | `运行 /opsx:apply 开始实现` | 微小差异 |

**合并策略：以 propose 版本为基准（更完整），ff-change skill 的 instructions 字段改为复用 propose 的同一 instructions 常量。**
