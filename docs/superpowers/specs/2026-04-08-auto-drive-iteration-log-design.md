# 设计文档：auto-drive 迭代记录机制

**日期**：2026-04-08  
**状态**：已批准

---

## 背景

`openspec-auto-drive` 当前没有持久化记录机制。每轮迭代的量化指标、方案决策、verify 发现等数据只存在于对话上下文中，对话结束即丢失。需要为每轮迭代写入独立记录文件，并在完成时生成摘要。

---

## 存储位置

```
.aiknowledge/auto-drive/
└── <change-name>/
    ├── iter-01.md
    ├── iter-02.md
    ├── iter-N.md
    └── summary.md     ← 完成时生成
```

选择 `.aiknowledge/auto-drive/` 而非变更目录，原因：跨变更可查，与 pitfalls 知识库同级，归档变更时不会带走日志。

---

## iter-N.md 格式

```markdown
# 第 N 轮迭代

**时间**：YYYY-MM-DD HH:MM  
**量化指标**：目标值 X，本轮实测 Y，差距 Z

## 本轮方案

explore/plan 阶段收敛的核心决策（1-3 句）

## verify 发现

- [CRITICAL/WARNING/SUGGESTION] 问题描述

## 决策

达标 / 继续下一轮 / 卡住回退

## 卡住原因（如有）

说明连续 2 轮无改进的表现和回退策略
```

**命名规则**：`iter-01.md`、`iter-02.md`……始终两位数补零，便于排序。

**写入时机**：Phase 7（决策）完成后立即写入，不等到下一轮开始。

---

## summary.md 格式

```markdown
# Auto-Drive 摘要：<change-name>

**总轮次**：N  
**最终状态**：达标 / 超出迭代上限 / 手动停止  
**目标**：<用户设定的优化目标>  
**量化标准**：<达标条件>

## 指标趋势

| 轮次 | 实测值 | 达标 |
|------|--------|------|
| 1    | Y1     | ✗    |
| 2    | Y2     | ✓    |

## 关键转折

- 第 N 轮：<什么决策带来了突破或停滞>

## 参考文件

- [第 1 轮](iter-01.md)
- [第 2 轮](iter-02.md)
```

**写入时机**：退出契约触发时（达标、超出上限、手动停止），汇总所有 iter 文件后生成。

---

## auto-drive.ts 改动范围

| 位置 | 改动说明 |
|---|---|
| 新增 `## 记录机制` 节 | 描述目录结构、文件命名规则、写入时机 |
| `Phase 6 - 度量` 末尾 | 度量完成后写 iter-N.md，记录指标、方案、verify 发现、决策 |
| `## 退出契约` | 生成 summary.md，输出路径提示用户 |

## 不变部分

- 引擎循环的 7 个阶段逻辑不变
- 门控级别（none / design / all）不变
- 卡住恢复机制不变
- 完成条件不变
