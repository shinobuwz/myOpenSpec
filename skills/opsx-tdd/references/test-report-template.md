# Test Report Template

`test-report.md` 是变更级实时测试留档文件，必须追加写入，禁止覆盖已有记录。

```md
# Test Report: <change-name>

## Task N: <描述>

### 🔴 红阶段
- **时间戳**：YYYY-MM-DD HH:MM
- **失败测试列表**：
  - `<test-case-name>` — `<ErrorType>`: <简要归因>（产品代码未实现 / 断言失败 / 测试代码错误）
- **假红修正**：<无 | 修正记录>

### 🟢 绿阶段
- **时间戳**：YYYY-MM-DD HH:MM
- **通过测试列表**：
  - `<test-case-name>` ✅
- **变更摘要**：
  - `<file-path>` — <新增|修改|删除> — <一句话意图>
- **测试变更记录**：<无测试修改 | 逐条记录>
- **验收标准覆盖对应表**：

| 验收标准 (AC) | 测试用例名称 | 通过状态 |
|--------------|------------|---------|
| ACN: ... | `<test-name>` | ✅ |

### ♻️ 重构阶段
- **时间戳**：YYYY-MM-DD HH:MM
- **通过测试列表**：
  - `<test-case-name>` ✅
- **变更摘要**：
  - `<file-path>` — <新增|修改|删除> — <一句话意图>
- **覆盖率**：xx%（命令：`<coverage-command>`）
```

无法提供数字覆盖率时：

```md
- **覆盖率**：N/A（<不可用原因>；补救：<下一步命令或不适用说明>）
```

不要写裸 `N/A`。

## `[manual]`

```md
### ⏳ 待人工验证
- [ ] [manual] <验收标准内容> — 验证方法：<方法描述>
```
