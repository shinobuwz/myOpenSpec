# TDD Anti Patterns

以下行为禁止：

1. 先写产品代码再补测试。
2. 写了测试但跳过红色阶段，测试一开始就通过。
3. 一次性写大量测试而不逐个循环。
4. 重构时不运行测试。
5. 用 mock 替代真实行为测试导致假阳性。

## 常见借口

| 借口 | 正确做法 |
|------|----------|
| "这个函数太简单了，不需要测试" | 简单函数也需要测试 |
| "我先写代码理清思路，再补测试" | 用测试理清思路 |
| "测试框架还没配好" | 先配好测试框架 |
| "这是 UI 代码，不好测试" | 分离逻辑层，逻辑层必须测试 |
| "时间不够写测试了" | 没有测试的代码不算完成 |
