# Implement Execution Rules

## 每个 task

1. 读取 design/specs 中对应部分。
2. 校验依赖 task 已完成。
3. 按执行方式标签执行：
   - `[test-first]`：红→绿→重构，每阶段追加 `test-report.md`
   - `[characterization-first]`：先固化旧行为，再修改代码，同样追加三阶段留档
   - `[direct]`：纯配置/脚手架直接执行
4. 确保每条非 `[manual]` 验收标准有测试、验证命令或无需测试理由。
5. `[manual]` 项写入待人工验证清单；已人工验证才勾选。
6. 每完成一个 task，立即把顶层任务 `[ ]` 改为 `[x]`，并把有证据支撑的验收标准 `[ ]` 改为 `[x]`。
7. 没有证据的验收标准不得勾选。
8. 禁止所有任务完成后才补写 `test-report.md`。

## 暂停条件

- 任务描述不清晰。
- 设计遗漏。
- 任务之间有未记录依赖。
- 技术问题超出当前 change 范围。
