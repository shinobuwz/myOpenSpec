# Review Risk Taxonomy

`opsx-review` 是发布风险拦截器，不是代码风格顾问。只报告有证据的真实风险。

## 审查类别

| 类别 | 关注点 |
|------|--------|
| verify drift | 明显需求遗漏、范围外实现、任务状态与代码证据不一致；输出 `VERIFY_DRIFT` 并回 `opsx-verify` |
| 逻辑错误 | 空指针、越界、死循环、条件反转、off-by-one |
| 资源泄漏 | 内存、文件句柄、网络连接、锁、订阅；含异常路径释放 |
| 并发问题 | 数据竞争、死锁、竞态条件；需追踪读写线程并确认无同步 |
| 安全问题 | 注入、越权、敏感信息泄露、不安全反序列化 |
| API 兼容性 | 接口签名或行为变更，调用方是否同步 |
| 错误处理 | 异常吞没、错误与空值混淆、缺少错误反馈 |
| 业务逻辑 | 条件判断、状态转换、边界行为是否符合设计意图 |
| 异步安全 | 上下文失效、过期结果覆盖、重复触发 |

## 禁止报告

- 格式类：缩进、对齐、import 顺序、注释格式。
- 风格偏好：命名争论、语法糖偏好、early return vs 嵌套。
- 假设性优化：没有调用路径和量化数据的缓存或数据结构建议。
- 超出项目版本的特性建议。

## 分级

### CRITICAL

必须同时满足：

1. 可触发：当前环境下能实际触发。
2. 无保护：没有 guard、try/catch、锁、条件编译等外部防护。
3. 严重后果：导致崩溃、数据损坏、安全漏洞或破坏性兼容问题。

### WARNING

- 性能退化且有量化依据。
- 数据源不一致。
- 状态不闭环。
- 删除代码移除了必要副作用。
- 新增 TODO/FIXME。

### SUGGESTION

- diff 中新增代码的 magic number、过长函数等局部清晰度问题。
- 能指出调用路径但缺少量化数据的潜在性能隐患。

## Pitfalls 使用

读取 `.aiknowledge/pitfalls/` 后，仅把 active 条目作为高权重核查项。stale 只作线索，superseded/deprecated 默认不作为依据。命中时在 issue 中引用 pitfall ID；如果 diff 证明 active 条目失效，输出 knowledge drift 建议归档前更新知识库。
